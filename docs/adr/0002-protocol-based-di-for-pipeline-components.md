# Protocol-based DI for pipeline components

The pipeline has four pluggable seams: file fetching (`FileFetcher`), metadata extraction (`MetadataExtractor`), structured persistence (`StructuredSink`), and unstructured persistence (`UnstructuredSink`). Each is defined as a `typing.Protocol` in `data_index/protocols.py`; concrete implementations live in `data_index/impl/`. Protocols were chosen over ABCs because they enable structural subtyping — third-party or test implementations need no inheritance. The `FileFetcher` operates at batch level (receives validated URIs, returns a Manifest DataFrame) rather than per-file, to allow implementations to choose their own fetch strategy (sequential, concurrent, bulk). `MetadataExtractor` is a single combined protocol (not split into structured/unstructured) to avoid opening the same `xarray.Dataset` twice. Both sinks are separate protocols because they have different data shapes, different target stores, and independent replacement trajectories.

# Hash join key with unstructured contract parity

We will use `ObjectReference.hash` (SHA-256 of `bucket`, `key`, `version_id`) as the sink join/upsert key for both structured and unstructured writes, while keeping `bucket`, `key`, and `version_id` as required explicit identity fields in all contracts and schemas. To align sink interfaces, unstructured writes move from `dict[versioned_uri, metadata]` to `list[UnstructuredMetadata]`, where `UnstructuredMetadata` mirrors the structured contract pattern (dataclass schema source-of-truth, schema versioning, top-level `file_format`, and `metadata` dict payload), and intermediate handle/cache abstractions are removed in favor of in-memory row handoff between transform and load. This is chosen to remove current contract drift, avoid sink-local re-derivation/parsing, and make identity semantics consistent across both sink families; Arrow-table transform/load handoff and local Parquet sink parity are explicitly deferred.

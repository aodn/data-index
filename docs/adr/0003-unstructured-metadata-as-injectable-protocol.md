# UnstructuredMetadata as injectable callable with load interface

Unstructured metadata dicts are potentially large and must not accumulate in memory across parallel `_transform_single` tasks. Rather than passing a raw `dict` through `ExtractionResult`, `unstructured_metadata` is typed as an `UnstructuredMetadata` protocol with a single `load() -> dict` method. A factory callable `metadata_factory: Callable[[str, dict], UnstructuredMetadata]` is injected into `transform` and `_transform_single`; calling `metadata_factory(s3_uri, data)` produces a persisted instance immediately. The default is `DiskCachedUnstructuredMetadata`, which writes to diskcache in `__init__` and releases the dict from memory; `InMemoryUnstructuredMetadata` is available for testing. The `UnstructuredSink` protocol and `load` receive lightweight protocol objects and call `.load()` only when writing to the final destination.

Alternatives considered: a `create` classmethod on the protocol (protocols can't enforce constructor signatures, so a `Callable` type hint is both simpler and more precise); passing `cache_path: Path` directly into `_transform_single` (couples the task to diskcache, non-injectable); keeping `unstructured_metadata: dict` in `ExtractionResult` and writing at end of `transform` (holds all dicts in memory until all parallel tasks complete, which was the problem being solved).
